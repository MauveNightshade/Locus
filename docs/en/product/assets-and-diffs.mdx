Locus organizes Unity assets as graph-level working objects and provides semantic diff and merge views for key asset types that are closer to Unity structure.

## Asset Database and Reference Graph

The current implementation already covers:

- Scanning `Assets` and `Packages`
- Parsing `.meta`, YAML assets, and script information
- Building asset nodes and reference edges
- Providing dependency lookup, reverse reference lookup, GUID resolution, and path resolution
- Asset risk reports
- Detection of broken references, `Missing Script`, parse failures, and duplicate GUIDs
- Incremental index updates through a file system watcher

This allows Locus to answer many high-value questions in Unity projects:

- Who references this Prefab
- Which assets will be affected by a ScriptableObject change
- Where duplicate GUID risks exist in the current project
- Which assets already contain Missing Script

## Semantic Diff and Merge

The diff and merge modules in the repository already cover:

- Scene hierarchy and Inspector structure views
- Semantic target construction for `PrefabInstance`
- Grouped field presentation for common objects such as materials and particle systems
- Dual views for semantic diff and text diff
- Merge semantic sessions and target materialization for Unity assets
- Conflict-resolution workflows and merge panels

## Suitable Usage Scenarios

- Asset troubleshooting and impact analysis
- Review of Scene, Prefab, Material, and ParticleSystem changes
- Unity asset conflict handling and merge assistance

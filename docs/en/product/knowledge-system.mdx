Locus turns knowledge into an independent workspace and clearly divides project knowledge into four categories, making long-term maintenance, unified retrieval, and stable injection easier.

## The Four Knowledge Semantics

- `Design`: project design and direction
- `Memory`: long-term memory and user preferences
- `Skill`: standard processes and SOP
- `Reference`: external materials

## Current Capabilities

- Unified knowledge tree and directory configuration
- Full-text retrieval and vector retrieval working in parallel
- Injection preview to inspect which knowledge blocks the current Agent will actually receive
- Local embedding runtime and remote embedding endpoint configuration
- Unity official documentation import
- Feishu knowledge-base import
- External Reference directory integration

## Direct Value for Teams

This set of capabilities is suitable for bringing official documentation, project conventions, Skill workflows, and project memory into one searchable and injectable system that the Agent can use, reducing repeated context loss.

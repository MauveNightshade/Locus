From the current repository snapshot, Locus already has the skeleton of a complete desktop workspace, the underlying runtime, and a portion of its stability work.

## Maturity Signals at the Code Level

- The main frontend is organized around six workspaces
- The Rust backend has been split into independent modules such as `agent / asset_db / diff / merge / knowledge_index / unity_bridge / llm / session / vcs`
- The Unity plugin already has an independent bridge implementation
- There are currently 93 test files under `src/__tests__`
- The session persistence schema has clearly evolved to `v10`, with explicit migration logic retained

## Teams It Fits Better

- Develop games with Unity
- Use Git to manage the project
- Want to truly embed AI into the daily development workflow
- Need to manage project knowledge, conventions, and external references in one place

## Current Product Focus

- Unity first
- Git first
- Local desktop workflow first
- Long-term project collaboration oriented

## Implementation Basis

The judgments in this page mainly come from the following modules in the current repository:

- Application shell: `src/App.vue`
- Session and Agent UI: `src/components/ChatView.vue`, `src/components/AgentView.vue`
- Asset workspace: `src/components/AssetView.vue`, `src-tauri/src/asset_db/*`, `src-tauri/src/commands/asset.rs`
- Collaboration workspace: `src/components/CollabView.vue`, `src-tauri/src/merge/*`, `src-tauri/src/diff/*`
- Knowledge workspace: `src/components/KnowledgeView.vue`, `src-tauri/src/knowledge_index/*`, `src-tauri/src/knowledge_store.rs`
- Unity bridge: `src-tauri/src/unity_bridge/*`, `locus_unity/Editor/LocusBridge*.cs`
- Models and runtime: `src/stores/model.ts`, `src-tauri/src/llm/*`, `src-tauri/src/auth/*`
- Session persistence and migration: `src-tauri/src/session/store.rs`

Locus connects the file layer, editor layer, and compilation layer into one complete chain. This chain determines whether the Agent can truly participate in actual Unity project development.

## Key Capabilities Already Covered

- Detect Unity Editor connection state and distinguish `editing / playing / playing_paused / disconnected`
- Select Unity assets from the desktop side and locate them in the Editor
- Execute C# code on the Unity side
- Request recompilation after script changes and wait for domain reload to complete
- Batch-import changed assets before recompilation
- Perform safety checks in Play Mode, on disconnect, and under inconsistent state

## Why This Chain Matters

Many real tasks are not just about editing files. They usually involve a full execution chain:

1. Modify scripts or assets.
2. Make Unity perceive those changes correctly.
3. Wait for compile results or domain reload.
4. Return to the Editor for continued validation.

Locus has already turned this chain into a product capability, so it is closer to a real Unity development loop.

## Tasks It Can Take On Well

- Request Unity recompilation immediately after script changes
- Trigger import after batch-processing asset changes
- Handle state awareness between runtime and edit mode
- Send error messages, state changes, and validation results directly back into the session

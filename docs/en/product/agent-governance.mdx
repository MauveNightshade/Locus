Locus is built for sustained use rather than one-off Q&A. For that reason, it turns models, rules, tool confirmation, and runtime visualization into an independent governance layer.

## Current Governance Capabilities

- Multi-model access: OpenRouter, Claude subscription, Claude Agent SDK, OpenAI Codex, custom endpoints
- Plan mode and Build mode
- Sub-agent / task tools
- Tool execution confirmation and batch confirmation
- Undo tracking for file changes in each round
- Raw context export
- Visual dashboard for Agent system prompt, env, rules, runtime injection, and tool schema

## What Problems This Set of Capabilities Solves

- Let teams control the Agent's default behavior and boundaries
- Make tool calls and file changes reviewable and reversible
- Make model selection, rule injection, and context composition more transparent in long-term projects

## Teams It Fits Better

- Already using multiple models in parallel
- Need to inspect the Agent runtime process
- Need to bring rules and knowledge into a stable governance workflow

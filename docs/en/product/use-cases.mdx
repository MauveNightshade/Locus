The direct value of Locus comes from three areas: reducing context switching, improving the quality of Unity problem-solving, and fitting long-term project use.

## Direct Value

### Less Context Switching

Developers can complete Unity, IDE, documentation, Wiki, and Git-related work inside one unified desktop workspace.

### Better Fit for Unity Project Problem Types

- Analyze asset dependencies and impact scope
- Handle the Unity recompilation loop after script changes
- Review Scene / Prefab / Material / ParticleSystem changes
- Find Missing Script, broken references, and duplicate GUIDs
- Bring Unity official documentation and team knowledge into the same retrieval entry point

### Better Fit for Long-Term Projects

Locus has already made knowledge structure, session persistence, schema migration, context export, and undo tracking into product infrastructure.

## Typical Use Cases

### Fix a Real Feature in a Unity Project

- Read related scripts and Prefabs
- Check asset references
- Modify code
- Request Unity recompilation
- Send compiler errors or domain reload state back into the session

### Troubleshoot Asset Problems

- Search for the target asset
- Review dependencies and reverse references
- Open asset risk reports
- Inspect Missing Script, broken references, and duplicate GUIDs

### Let the Agent Work with Project Knowledge

- Inject project design constraints
- Inject engineering conventions
- Inject common workflows
- Inject Unity official documentation and external knowledge bases

### Review and Merge Complex Asset Changes

When team members modify Scenes, Prefabs, and Materials at the same time, Locus can provide diff and merge views that are closer to Unity semantics and help developers understand the changes more quickly.
